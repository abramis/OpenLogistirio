# Open Logistirio

Το Open Logistirio είναι ένα ανεξάρτητο, ανοιχτού κώδικα ERP για ελληνικά λογιστικά γραφεία.
Υποστηρίζει πολλαπλούς πελάτες, παραστατικά, λογιστική παρακολούθηση, ΦΠΑ, συμφωνία myDATA,
πάγια, υποχρεώσεις, εισαγωγές, δικαιολογητικά, audit log και αντίγραφα ασφαλείας.

> **Χρησιμοποιείς Windows σε λογιστικό γραφείο;** Ακολούθησε μόνο την ενότητα
> [1. Κανονική χρήση σε Windows](#1-κανονική-χρήση-σε-windows-για-λογιστικό-γραφείο).
> Οι ενότητες για Linux server και development δεν χρειάζονται για την κανονική χρήση.

Η τρέχουσα έκδοση είναι **πιλοτική** και προορίζεται για ελεγχόμενη δοκιμή από λογιστικά
γραφεία. Δεν περιλαμβάνει ακόμη μισθοδοσία, ΑΠΔ/ΕΡΓΑΝΗ, δηλώσεις εισοδήματος ή επίσημη
υποβολή δηλώσεων. Δεν αντικαθιστά ακόμη όλα τα συστήματα και τις διαδικασίες του γραφείου.

Οι εγγραφές προς το παραγωγικό περιβάλλον της ΑΑΔΕ είναι απενεργοποιημένες από προεπιλογή.
Μην τις ενεργοποιήσεις για δοκιμή. Χρησιμοποίησε αρχικά δοκιμαστικά ή ανωνυμοποιημένα δεδομένα,
κράτησε την υπάρχουσα διαδικασία υποβολών και πάρε ανεξάρτητο αντίγραφο ασφαλείας πριν βασιστείς
στην εφαρμογή για πραγματικά δεδομένα.

## 1. Κανονική χρήση σε Windows για λογιστικό γραφείο

Αυτή είναι η προτεινόμενη εγκατάσταση για ένα λογιστικό γραφείο που θέλει να χρησιμοποιήσει ή
να δοκιμάσει την εφαρμογή σε ένα Windows PC χωρίς δικό του τμήμα IT.

Από προεπιλογή η εφαρμογή είναι διαθέσιμη μόνο σε αυτό το PC. Για πρόσβαση από πολλά PC ή μέσω
Internet χρειάζεται η τεχνική εγκατάσταση Linux server της ενότητας 2.

Η εγκατάσταση χρησιμοποιεί τα **production containers** της εφαρμογής. Ξεκινά με **κενή βάση**
και δεν δημιουργεί demo γραφείο, demo χρήστες, demo πελάτες ή demo παραστατικά.

Δεν χρειάζεται να εγκαταστήσεις Node.js, Angular, MySQL ή Redis. Δεν χρειάζεται να ανοίξεις
terminal, να γράψεις εντολές ή να επεξεργαστείς αρχεία `.env`.

### Πρώτη εγκατάσταση

1. Εγκατέστησε το Docker Desktop για Windows με WSL 2/Linux containers και περίμενε μέχρι να
   δείξει ότι λειτουργεί.
2. Άνοιξε τη [σελίδα Releases](https://github.com/abramis/OpenLogistirio/releases) και κατέβασε
   το `OpenLogistirio-Windows-<έκδοση>.zip` της έκδοσης που θέλεις να δοκιμάσεις. Αν δεν υπάρχει
   ξεχωριστό Windows ZIP, κατέβασε το **Source code (zip)** της ίδιας έκδοσης.
3. Κάνε αποσυμπίεση σε έναν μόνιμο φάκελο, για παράδειγμα `C:\OpenLogistirio`. Μην τρέξεις την
   εγκατάσταση μέσα από το ZIP ή από προσωρινό φάκελο.
4. Κάνε διπλό κλικ στο `INSTALL-WINDOWS.cmd`.
5. Περίμενε να ανοίξει στον browser η σελίδα **Αρχική ρύθμιση** και συμπλήρωσε:
   - επωνυμία, ΑΦΜ και στοιχεία επικοινωνίας του λογιστικού γραφείου,
   - ονοματεπώνυμο και email του πρώτου διαχειριστή,
   - έναν προσωπικό κωδικό τουλάχιστον 14 χαρακτήρων, με πεζό, κεφαλαίο, αριθμό και σύμβολο.
6. Ολοκλήρωσε τη φόρμα. Η αρχική ρύθμιση επιτρέπεται μόνο μία φορά και μετά ανοίγει η εφαρμογή
   με τον πραγματικό λογαριασμό διαχειριστή του γραφείου.

Το installer δημιουργεί μόνο του τα τεχνικά secrets, ξεκινά τις υπηρεσίες, εφαρμόζει τις
απαραίτητες αλλαγές βάσης και ανοίγει τη σωστή τοπική διεύθυνση. Δεν αποθηκεύει τον κωδικό του
διαχειριστή σε αρχείο ρυθμίσεων.

Μετά την πρώτη σύνδεση δημιούργησε από τους **Χρήστες** έναν δεύτερο, έμπιστο διαχειριστή και
φύλαξε τους δύο προσωπικούς κωδικούς σε password manager. Η πιλοτική έκδοση δεν στέλνει ακόμη
email επαναφοράς κωδικού, ώστε να μη δίνει ψεύτικη επιλογή ανάκτησης χωρίς ρυθμισμένη υπηρεσία
email.

### Καθημερινή εκκίνηση και διακοπή

- Για εκκίνηση, βεβαιώσου ότι λειτουργεί το Docker Desktop και κάνε διπλό κλικ στο
  `START-WINDOWS.cmd`.
- Για διακοπή, κάνε διπλό κλικ στο `STOP-WINDOWS.cmd`.
- Το `INSTALL-WINDOWS.cmd` χρειάζεται στην πρώτη εγκατάσταση και όταν οι οδηγίες μιας νέας
  έκδοσης ζητούν αναβάθμιση. Δεν χρειάζεται στην καθημερινή εκκίνηση.

Η διακοπή ή επανεκκίνηση των containers δεν διαγράφει τα δεδομένα. Η βάση, τα δικαιολογητικά και
τα τοπικά backups αποθηκεύονται σε σταθερά Docker volumes, ξεχωριστά από τα αρχεία του ZIP.

### Αντίγραφα ασφαλείας και ενημέρωση

Από την επιλογή **Αντίγραφα ασφαλείας** της εφαρμογής ο διαχειριστής μπορεί να δημιουργήσει και
να κατεβάσει backup της βάσης. Κράτησε αντίγραφο και σε διαφορετικό, προστατευμένο μέσο. Τα
backups που βρίσκονται μόνο στο ίδιο PC ή μόνο μέσα στο Docker δεν προστατεύουν από βλάβη ή
απώλεια του υπολογιστή. Τα δικαιολογητικά χρειάζονται επίσης ανεξάρτητο αντίγραφο.

Πριν από ενημέρωση:

1. Δημιούργησε και κατέβασε νέο backup.
2. Διάβασε τις σημειώσεις της συγκεκριμένης έκδοσης για τυχόν ειδικές οδηγίες.
3. Σταμάτησε την εφαρμογή με `STOP-WINDOWS.cmd`.
4. Αποσυμπίεσε τη νέα έκδοση σε μόνιμο φάκελο και κάνε διπλό κλικ στο `INSTALL-WINDOWS.cmd` της
   νέας έκδοσης, από τον ίδιο λογαριασμό Windows που έκανε την αρχική εγκατάσταση.

Το installer βρίσκει τις ασφαλείς ρυθμίσεις του ίδιου Windows λογαριασμού, επαναχρησιμοποιεί τα
σταθερά volumes και δεν ξανανοίγει την αρχική φόρμα όταν υπάρχει ήδη διαχειριστής. Μην διαγράψεις
ή μετονομάσεις χειροκίνητα τα Docker volumes και μην χρησιμοποιήσεις την επιλογή επαναφοράς
εργοστασιακών ρυθμίσεων του Docker Desktop χωρίς επιβεβαιωμένο εξωτερικό backup. Επιστροφή σε
παλαιότερη έκδοση μπορεί να μην είναι δυνατή μετά από migration βάσης.

Τα στοιχεία σύνδεσης ΑΑΔΕ δεν είναι απαραίτητα για την πρώτη εγκατάσταση. Η προαιρετική και
προχωρημένη παραμετροποίησή τους περιγράφεται στον [οδηγό ΑΑΔΕ](docs/aade-setup.md).

> Η ύπαρξη Windows ZIP δεν αποτελεί από μόνη της επιβεβαίωση ότι η συγκεκριμένη έκδοση πέρασε
> πραγματική δοκιμή σε Windows. Έλεγξε τις σημειώσεις του release πριν χρησιμοποιήσεις πραγματικά
> δεδομένα.

## 2. Εγκατάσταση σε Linux server — μόνο για τεχνικό

Αυτή η διαδρομή αφορά κεντρικό server, πρόσβαση από πολλά PC ή πρόσβαση μέσω δικτύου/Internet.
Απαιτεί Linux, HTTPS, διαχείριση secrets, monitoring και ανεξάρτητα off-site backups. Δεν είναι η
διαδρομή που πρέπει να ακολουθήσει ένας λογιστής για εγκατάσταση στο Windows PC του.

Οι τεχνικές οδηγίες βρίσκονται στο [Linux production runbook](docs/production-runbook.md).

## 3. Development — μόνο για προγραμματιστές

Οι παρακάτω τρόποι χρησιμοποιούν development services και demo δεδομένα. Δεν προορίζονται για
κανονικά δεδομένα λογιστικού γραφείου.

1. **Docker Compose development — προτεινόμενο για γρήγορη εκκίνηση:** χρειάζεται μόνο Docker και
   σηκώνει development API, web, MySQL και Redis.
2. **Τοπικό Node.js development:** για προγραμματιστές που θέλουν API και web έξω από containers
   και έχουν ήδη Node.js, MySQL/MariaDB και Redis.

Οι εντολές, οι θύρες και η αναλυτική παραμετροποίηση βρίσκονται στον
[οδηγό development](docs/development.md).
