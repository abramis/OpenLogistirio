# Open Logistirio UI system

Το UI είναι εργαλείο καθημερινής εργασίας ελληνικού λογιστικού γραφείου. Προτεραιότητα έχουν η
γρήγορη σάρωση, η ασφαλής εκτέλεση ενεργειών και η χαμηλή οπτική κόπωση — όχι η μέγιστη ποσότητα
πληροφορίας στην πρώτη οθόνη.

## Βασική δομή σελίδας

Κάθε νέα σελίδα ακολουθεί την ίδια σειρά:

1. `page-header`: τίτλος, μία σύντομη περιγραφή και έως δύο κύριες ενέργειες.
2. alerts: μόνο τρέχον αποτέλεσμα ή πρόβλημα που απαιτεί προσοχή.
3. σύνοψη/KPIs: μόνο μετρήσεις που αλλάζουν την επόμενη ενέργεια του χρήστη.
4. `filter-bar` ή toolbar: scope, περίοδος και φίλτρα σε μία ήρεμη επιφάνεια.
5. κύριο workspace: λίστα, workpaper ή φόρμα.
6. προχωρημένες ρυθμίσεις: σε `details.disclosure`, κλειστές αρχικά.

Δεν αναμειγνύουμε φίλτρα, καταχώριση και αποτελέσματα μέσα στην ίδια αδιαίρετη card.

## Χρώμα και επιφάνειες

- `--bg`: καμβάς εφαρμογής. Δεν γίνεται λευκός.
- `--surface`: ενεργό περιεχόμενο, πίνακες και cards.
- `--surface-2`: headers και δευτερεύουσες περιοχές.
- `--surface-3`: table headers και disabled/read-only περιοχές.
- `--surface-tint`: φίλτρα, totals και περιοχές ελέγχου.
- `--primary`: κύρια ενέργεια και ενεργή πλοήγηση.
- semantic χρώματα (`--ok`, `--warn`, `--err`) χρησιμοποιούνται μόνο για πραγματική κατάσταση,
  όχι για διακόσμηση.

Ανά οθόνη επιτρέπεται μία κύρια χρωματική έμφαση. Τα λευκά blocks πρέπει να περιβάλλονται από
ουδέτερο καμβά ή tinted section ώστε να παραμένουν διακριτά χωρίς βαριές σκιές.

## Φόρμες

- Οι φόρμες χωρίζονται σε αριθμημένα ή σαφώς ονομασμένα `form-section` blocks.
- Ένα section περιέχει κατά κανόνα 3–8 σχετικά πεδία. Μεγαλύτερο σύνολο διασπάται.
- Τα υποχρεωτικά πεδία και τα συχνά χρησιμοποιούμενα εμφανίζονται πρώτα.
- myDATA classifications, provider signatures, POS/ECR metadata, ειδικοί φόροι και σπάνιες
  εξαιρέσεις μπαίνουν σε disclosure.
- Η κύρια ενέργεια αποθήκευσης βρίσκεται στο τέλος και παραμένει μία. Δευτερεύουσες ενέργειες
  χρησιμοποιούν `btn-secondary` ή `btn-ghost`.
- Validation εμφανίζεται δίπλα στο σχετικό πεδίο. Το γενικό error κρατά μόνο τη σύνοψη.

## Πίνακες και ουρές εργασίας

- Ο πίνακας έχει σαφή τίτλο και εξηγεί τι είναι η λίστα, όχι μόνο τα ονόματα στηλών.
- Πρώτες εμφανίζονται οι εγγραφές που απαιτούν χειρισμό.
- Ποσά στοιχίζονται δεξιά, status αποδίδεται με badge και η βασική ταυτότητα έχει δεύτερη γραμμή
  (`small`) για ΑΦΜ, MARK ή άλλο αναγνωριστικό.
- Πάνω από τρεις row actions ομαδοποιούνται ή μεταφέρονται σε detail view/disclosure.
- Empty state εξηγεί αν «δεν υπάρχει δεδομένο» ή «δεν βρέθηκε με τα φίλτρα».

## Γλώσσα και ασφάλεια

- Τα labels είναι ελληνικά και χρησιμοποιούν λογιστικούς όρους που αναγνωρίζει ελληνικό γραφείο.
- Αγγλικά identifiers εμφανίζονται μόνο όταν είναι επίσημο πεδίο AADE/provider ή τεχνικό reference.
- Ενέργειες ΑΑΔΕ δηλώνουν ρητά αν είναι ανάγνωση ή εγγραφή.
- Production write, cancellation και μαζικές μη αναστρέψιμες ενέργειες χρειάζονται ξεχωριστή,
  σαφή επιβεβαίωση. Δεν κρύβονται ανάμεσα σε ουδέτερα κουμπιά.

## Responsive και accessibility

- Στα 760px τα grids γίνονται μονής στήλης και οι actions τυλίγονται χωρίς οριζόντιο scroll.
- Όλα τα controls έχουν ορατό focus state, label και επαρκές clickable area.
- Δεν χρησιμοποιούμε μόνο χρώμα για να αποδώσουμε κατάσταση: badge text ή icon συνοδεύει το χρώμα.
- Material icons που λειτουργούν ως μοναδικό label χρειάζονται `aria-label`.

## Checklist νέας οθόνης

- Ο χρήστης καταλαβαίνει μέσα σε λίγα δευτερόλεπτα ποια είναι η επόμενη ενέργεια;
- Υπάρχει μόνο μία κύρια ενέργεια ανά περιοχή;
- Τα σπάνια πεδία είναι κρυμμένα μέχρι να χρειαστούν;
- Υπάρχει ουδέτερος διαχωρισμός μεταξύ scope, input και results;
- Οι καταστάσεις loading, empty, success και error είναι σαφείς;
- Η οθόνη δουλεύει με keyboard και σε στενό viewport;
